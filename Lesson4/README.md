# 1. Цель и задачи практической работы

Целью данной работы являлось изучение механизмов асинхронного выполнения задач в Android-приложениях, а также получение практических навыков работы с потоками, сервисами и средствами фоновой обработки данных.

В процессе выполнения были рассмотрены и реализованы следующие задачи:

- исследование принципов работы многопоточности в Android;
- изучение особенностей главного UI-потока и причин возникновения ошибок ANR;
- работа с классами `Thread` и `Runnable` для запуска фоновых вычислений;
- организация взаимодействия между рабочими потоками и пользовательским интерфейсом;
- применение `Handler` и `Looper` для обмена сообщениями между потоками;
- использование `AsyncTaskLoader` для асинхронной обработки данных;
- реализация шифрования и дешифрования информации с использованием алгоритма AES;
- создание фоновых сервисов и сервисов переднего плана;
- изучение возможностей `WorkManager` для выполнения гарантированных фоновых задач.

# 2. Структура проекта

Практическая работа была разделена на несколько независимых модулей, каждый из которых демонстрирует отдельный механизм фонового выполнения:

| Модуль | Назначение |
|---|---|
| `thread` | Работа с потоками и вычислениями |

| `work_manager` | Планирование фоновых задач |
| `MireaProject` | Интеграция WorkManager в основной проект |

# 3. Выполнение практической работы

## 3.1. Работа с потоками (`thread`)

В данном модуле была реализована работа с главным потоком приложения. Для удобства отладки было изменено имя текущего потока, после чего выполнена организация вычислений в отдельном фоне.

Пользователь вводит количество занятий и число учебных дней, после чего программа вычисляет среднее количество пар за день. Все вычисления происходят в отдельном потоке, благодаря чему интерфейс приложения остается отзывчивым.

### Основные особенности реализации:
- запуск фонового потока через `Thread`;
- выполнение вычислений вне UI-потока;
- обновление интерфейса при помощи `runOnUiThread`;
- логирование работы потоков.

## 3.2. Обновление интерфейса из фонового потока (`data_thread`)

В этом модуле были рассмотрены способы безопасного взаимодействия с пользовательским интерфейсом из фоновых потоков.

Были протестированы три механизма:
- `runOnUiThread`;
- `post`;
- `postDelayed`.

Приложение поэтапно изменяет текст на экране с различными задержками, демонстрируя последовательное выполнение задач.

Использование подобных механизмов особенно важно, поскольку прямое изменение элементов интерфейса из рабочего потока вызывает исключение `CalledFromWrongThreadException`.

## 3.3. Использование Looper и Handler (`looper`)

В рамках данного задания был реализован собственный поток с циклом обработки сообщений.

Класс `MyLooper` создаёт отдельный поток, который после запуска продолжает ожидать новые сообщения благодаря использованию:
- `Looper.prepare()`;
- `Looper.loop()`.

Пользователь вводит профессию и возраст, после чего данные отправляются в рабочий поток. Поток имитирует длительную операцию с задержкой и возвращает результат обратно в главный поток через `Handler`.

### Особенности реализации:
- обмен сообщениями между потоками;
- использование `Message` и `Bundle`;
- возврат результата в UI-поток.

## 3.4. Асинхронная обработка данных (`cryptoloader`)

В данном модуле был реализован загрузчик `AsyncTaskLoader`, выполняющий дешифрование данных в фоновом режиме.

Перед отправкой данных строка шифруется при помощи алгоритма AES-256. Далее зашифрованные данные и ключ передаются в загрузчик через `Bundle`, где выполняется обратное преобразование.

Использование `AsyncTaskLoader` позволяет:
- избежать блокировки интерфейса;
- сохранить результаты при изменении конфигурации устройства;
- предотвратить утечки памяти.

### Основные возможности:
- генерация AES-ключа;
- шифрование пользовательского текста;
- дешифрование данных в фоне;
- вывод результата через `Toast`.

## 3.5. Реализация Foreground Service (`service_app`)

В этом модуле был создан музыкальный сервис переднего плана.

Сервис продолжает работать даже после сворачивания приложения, поскольку запускается как `Foreground Service`.

Для корректной работы были:
- добавлены необходимые разрешения;
- создан `NotificationChannel`;
- настроено постоянное уведомление в панели уведомлений.

После нажатия кнопки запуска начинается воспроизведение музыкального файла, а при остановке сервис завершает работу.

### Основные особенности:
- использование `MediaPlayer`;
- запуск через `startForeground`;
- отображение уведомления;
- поддержка Android Oreo и выше.

В данной части практической работы была реализована система выполнения фоновых задач при помощи `WorkManager`.

Создан класс `UploadWorker`, выполняющий длительную операцию в фоновом режиме. Запуск задачи производится через `OneTimeWorkRequest`.

Также были настроены ограничения (`Constraints`), определяющие условия выполнения задачи.

Преимущества `WorkManager`:
- гарантированное выполнение задачи;

# 4. Заключение

В ходе выполнения практической работы были изучены основные способы организации асинхронной обработки данных в Android.

На практике были освоены:
- работа с потоками;
- обмен сообщениями между потоками;
- безопасное обновление UI;
- использование сервисов;
- применение WorkManager;
- асинхронная обработка и защита данных.

В результате стало понятно, что длительные операции не должны выполняться в главном потоке приложения, поскольку это ухудшает производительность и может привести к зависанию интерфейса.

Также было установлено, что:
- `Thread` и `Handler` подходят для простых фоновых операций;
- `Service` используется для длительных процессов;
- `WorkManager` является наиболее удобным инструментом для гарантированного выполнения задач в фоне.
