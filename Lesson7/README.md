# 2. Структура проекта

В проекте используются следующие модули:

1. `timeservice` — приложение для получения точного времени по сети.

# 3. Выполнение практической работы

# 3.1. Получение сетевого времени (модуль `timeservice`)

В модуле `timeservice` реализовано получение точного времени с удалённого сервера. Первоначально для этого использовалось подключение к `time.nist.gov` по сокету на порту `13`, однако в процессе тестирования было выявлено, что на некоторых устройствах этот сервис может возвращать пустой ответ. Поэтому в приложение был добавлен дополнительный надёжный сценарий получения времени через HTTPS.

В модуле были реализованы:

- работа с сокетами через `Socket`;
- чтение данных через `BufferedReader`;
- обработка сетевых ошибок и таймаутов;
- резервное получение времени по HTTP `Date`-заголовку;
- вывод даты и времени на экран.

При нажатии на кнопку приложение выполняет запрос к серверу времени и отображает результат в виде отдельных строк `Дата` и `Время`.

# 3.2. Определение внешнего IP и получение погоды (модуль `httpurlconnection`)

Во втором модуле реализована цепочка сетевых запросов через `HttpURLConnection`. Сначала приложение обращается к сервису `https://ipinfo.io/json`, получает внешний IP-адрес устройства и сопутствующие данные о местоположении. После этого из поля `loc` извлекаются координаты, которые передаются в сервис `Open-Meteo` для получения погодной информации.

В модуле были реализованы:

- HTTP GET-запросы через `HttpURLConnection`;
- чтение ответа сервера через `InputStream`;
- разбор JSON-ответа через `JSONObject`;
- вывод на экран данных:
  - внешний IP;
  - город;
  - регион;
  - страна;
  - провайдер;
  - почтовый индекс;
  - часовой пояс;
  - координаты;
- запрос текущей погоды по координатам;
- отображение температуры, скорости ветра и кода погоды.

Приложение также проверяет наличие интернет-соединения через `ConnectivityManager` и выводит сообщение пользователю, если сеть недоступна.

# 3.3. Аутентификация через Firebase (модуль `firebaseauth`)

В модуле `firebaseauth` реализована полноценная работа с Firebase Authentication. Пользователь может создать аккаунт по email и паролю, войти в приложение, выйти из аккаунта и отправить письмо для подтверждения электронной почты.

В процессе выполнения задания были изучены:

- подключение Firebase к Android-проекту;
- использование `google-services.json`;
- инициализация `FirebaseAuth`;
- регистрация пользователя через `createUserWithEmailAndPassword()`;
- вход пользователя через `signInWithEmailAndPassword()`;
- отправка письма подтверждения через `sendEmailVerification()`;
- отображение `UID` пользователя и статуса `verified`.

В модуле были добавлены:

- валидация введённых email и пароля;
- сохранение сессии пользователя между запусками приложения;
- обработка и вывод текстов ошибок Firebase;
- разрешение `INTERNET` в `AndroidManifest.xml`.

После успешной регистрации пользователь отображается в интерфейсе приложения, а также появляется в консоли Firebase Authentication.
